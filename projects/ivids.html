The Cinematic TV & Browser Player

Traditional streaming interfaces often work well with a mouse, but feel awkward on a TV. IVIDS focuses on the opposite: big-screen navigation, clear media cards, predictable remote-control movement, and an edge-to-edge viewing experience that makes a personal media collection feel like a polished entertainment hub.

It is useful when you want the same player concept across Android TV, Smart TV-style screens, mobile, and regular browsers. The interface adapts its layout and input behavior to the device, so content stays front and center instead of being buried behind desktop-style controls.

Technical Notes

IVIDS keeps the technical foundation lightweight so the interface can respond quickly to remote-control movement and platform changes.

Bespoke SPA Router

Navigation is driven by a custom Single Page Application (SPA) routing engine (router.js). HTML partials are fetched from the pages/ folder on demand, and page-specific JS modules are imported asynchronously (e.g., pages/home.js). This enables sub-millisecond page transitions and keeps memory footprints tiny.

D-Pad Focus Engine

To achieve a native TV feel, the Spatial Navigation system tracks visible .focusable DOM items. It maps hardware arrow key strokes to calculate geometric proximity, auto-scrolling focused cards into center-screen position and adapting focus paths depending on device orientation.

Platform Execution & Android Packages

The codebase compiles directly into a native Android application package for immediate deployment to Android devices or Android TVs, while remaining fully hostable via any lightweight HTTP server for browser access.

Technical Stack

  • Vanilla JavaScript (ES6+ Modules)
  • Spatial Navigation Engine
  • Custom SPA HTML Router
  • i18n Localization
  • Android Gradle / Kotlin Wrapper
  • CSS Grid & Flexbox

Getting Started

Install on Android: Transfer and install the pre-built application package to your Android device or Android TV to run the native media player.

Run via Web Server: Serve the assets/main/gui directory using any local web server (such as Python http.server or VS Code Live Server) to run the interface on any desktop or mobile browser.
