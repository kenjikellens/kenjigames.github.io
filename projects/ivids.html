The Cinematic TV & Browser Player

Traditional streaming interfaces are built primarily for desktop mouse clicks and mobile taps, leaving TV remote controllers feeling sluggish and disconnected. IVIDS was designed from the ground up to solve this. It provides a locked 60 FPS, edge-to-edge media experience optimized for Android TVs, Tizen Smart TVs, and standard web browsers alike.

Using a strict "zero-framework" standard, the interface adapts dynamically to the host platform. Whether running inside a native Android WebView or being served over standard web protocols, IVIDS coordinates user input, network caching, and visual layouts to place content front and center.

Spatial Navigation

Custom geometric focus manager optimized for TV remote D-pads and hardware navigation keys.

Universal Layouts

Adaptive structural scaling that adjusts between TV (Landscape) and Mobile (Portrait) viewports.

i18n Translation

Global translation module dynamically swap UI localizations from flat JSON files without reload.

Technical Architecture

By refusing the heavy overhead of large frameworks, IVIDS relies on core ES Modules and DOM observer patterns to maintain clean rendering pipelines and instant feedback loops.

Bespoke SPA Router

Navigation is driven by a custom Single Page Application (SPA) routing engine (router.js). HTML partials are fetched from the pages/ folder on demand, and page-specific JS modules are imported asynchronously (e.g., pages/home.js). This enables sub-millisecond page transitions and keeps memory footprints tiny.

D-Pad Focus Engine

To achieve a native TV feel, the Spatial Navigation system tracks visible .focusable DOM items. It maps hardware arrow key strokes to calculate geometric proximity, auto-scrolling focused cards into center-screen position and adapting focus paths depending on device orientation.

Platform Execution & Android Packages

The codebase compiles directly into a native Android application package for immediate deployment to Android devices or Android TVs, while remaining fully hostable via any lightweight HTTP server for browser access.

Getting Started

Install on Android: Transfer and install the pre-built application package to your Android device or Android TV to run the native media player.

Run via Web Server: Serve the assets/main/gui directory using any local web server (such as Python http.server or VS Code Live Server) to run the interface on any desktop or mobile browser.
