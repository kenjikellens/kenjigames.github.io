The app is made for practical downloading sessions: grab one song, convert a long playlist, or target only a specific range of tracks. Technical Notes

The product experience is supported by a dual-backend architecture. On the Python side, StartUp.py spins up a ThreadingHTTPServer with custom route handlers that serve static UI assets and expose REST/SSE endpoints for download orchestration. On the Electron side, main.js creates a frameless BrowserWindow and uses a secure contextBridge preload script to expose IPC channels between the renderer and the Node.js downloadManager.js backend.

Server-Sent Events Streaming

The Python server establishes long-lived SSE connections for download sessions. As yt-dlp processes tracks, the DownloadManager class parses stdout in real-time, extracting percentage values from [download] log lines and broadcasting them as structured SSE events. A mutex lock prevents concurrent download sessions on the same server instance.

Electron IPC Bridge

The desktop variant uses Electron's contextIsolation security model. The preload.js script exposes a tightly scoped window.electronAPI object with invoke/send channels for directory selection, download control, and window management — all without enabling raw nodeIntegration in the renderer.

Unified Runtime Detection

The shared web UI (renderer.js) performs runtime detection at startup: if window.electronAPI exists, it routes through IPC; otherwise it falls back to HTTP fetch and EventSource APIs targeting the Python server. This single-codebase approach eliminates UI duplication across platforms.

Getting Started

Python Server: Run python StartUp.py to spin up the local web server on port 8080. Your default browser opens automatically and is ready to process downloads.

Electron Desktop: Install dependencies with npm install, then launch with npm start. Build a portable Windows EXE via build.bat.

Android: Point a WebView client to the Python server URL (http://<your-pc-ip>:<port>) for mobile access.
