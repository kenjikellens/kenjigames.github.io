IVIDS YT Downloader

Premium YouTube audio downloader with glassmorphism UI, playlist support, and triple-runtime architecture.

IVIDS YT Downloader is a high-performance YouTube audio extraction tool accessible through three parallel pathways: a Python web server, an Electron desktop app, and an Android WebView wrapper. Paste any YouTube URL or full playlist, pick your audio format and quality, and let the concurrent download engine handle the rest.

Three Runtimes, One Interface

Most downloaders lock you into a single platform. IVIDS YT Downloader breaks that limitation with a unified glassmorphism web UI that runs identically across three execution modes: a lightweight Python HTTP server with REST API and Server-Sent Events streaming, a native Electron desktop application with secure IPC bridge, and a mobile Android WebView wrapper for on-the-go access.

The web UI automatically detects its host runtime — Electron IPC or Python SSE — and adapts its communication layer accordingly. This means developers maintain a single codebase while delivering native-feel experiences on Windows, Android, and any modern browser.

Technical Architecture

The application is built around a dual-backend architecture. On the Python side, StartUp.py spins up a ThreadingHTTPServer with custom route handlers that serve static UI assets and expose REST/SSE endpoints for download orchestration. On the Electron side, main.js creates a frameless BrowserWindow and uses a secure contextBridge preload script to expose IPC channels between the renderer and the Node.js downloadManager.js backend.

Server-Sent Events Streaming

The Python server establishes long-lived SSE connections for download sessions. As yt-dlp processes tracks, the DownloadManager class parses stdout in real-time, extracting percentage values from [download] log lines and broadcasting them as structured SSE events. A mutex lock prevents concurrent download sessions on the same server instance.

Electron IPC Bridge

The desktop variant uses Electron's contextIsolation security model. The preload.js script exposes a tightly scoped window.electronAPI object with invoke/send channels for directory selection, download control, and window management — all without enabling raw nodeIntegration in the renderer.

Unified Runtime Detection

The shared web UI (renderer.js) performs runtime detection at startup: if window.electronAPI exists, it routes through IPC; otherwise it falls back to HTTP fetch and EventSource APIs targeting the Python server. This single-codebase approach eliminates UI duplication across platforms.

Getting Started

Python Server: Run python StartUp.py to spin up the local web server on port 8080. Your default browser opens automatically and is ready to process downloads.

Electron Desktop: Install dependencies with npm install, then launch with npm start. Build a portable Windows EXE via build.bat.

Android: Point a WebView client to the Python server URL (http://<your-pc-ip>:<port>) for mobile access.
