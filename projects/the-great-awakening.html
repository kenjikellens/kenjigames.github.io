The Great Awakening

A premium, academic-grade research archive and institutional analysis platform.

The Great Awakening is a declassified dossier repository engineered as a high-performance SPA. Bypassing heavy client frameworks, it couples custom routing with a flat-file JS database, a persistent dark/light theme engine, a custom background canvas visualizer, and an integrated AI chat widget.

Authoritative Archival Exploration

Institutional analysis requires structural clarity and uninterrupted reading focus. The Great Awakening was designed to combine academic rigor with high-speed web delivery. The interface provides a distraction-free environment for inspecting documents, timelines, and entity relations.

To maintain speed and SEO compliance, the application is built entirely as a vanilla SPA. Navigation, dossier search indexing, chat assistance, and rendering models run directly in the browser, eliminating rendering lags and database connection latency.

Technical Engineering

The platform's vanilla Single Page Application framework ensures that files and sections load dynamically with zero third-party dependencies.

Hash-Based SPA Routing

A custom router (js/router.js) tracks hash modifications (e.g., #home, #dossiers, #dossier/slug). It dynamically loads fragment partials from pages/fragments/ and injects dossier HTML from pages/dossiers/ into the content shell, extracting styles to preserve unique layouts.

JSON Database Layer

A centralized flat-file database (data/dossiers.js) catalogs metadata for all research records. The Dossier Manager (js/dossier-manager.js) reads this file to build index lists, update the dynamic mega menu, and drive the local search engine.

AI Chat & Theme Utilities

An inline chat module (js/ai-chat.js) powers the helper drawer widget. User options and theme scales are governed by the theme-manager.js, saving selections to localStorage for seamless persistence.

Getting Started

Clone the source and serve the directory using a lightweight web server (e.g. python -m http.server 8000 or VS Code Live Server) to enable fetch() functionality for the dynamic dossier fragments.
